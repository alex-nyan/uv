# Reproduction — astral-sh/uv#6264 (Fix reflow of index on image load)

> **Issue:** [astral-sh/uv#6264](https://github.com/astral-sh/uv/issues/6264) — *"e.g. at https://docs.astral.sh/uv/ we reflow when the benchmark loads. We need to set a fixed height for the image."*

This folder is a **self-contained reproduction** of the layout shift (cumulative layout
shift / CLS) on the uv documentation homepage. It does **not** modify uv; it isolates the
exact markup from [`docs/index.md`](https://github.com/astral-sh/uv/blob/main/docs/index.md)
so the bug can be observed and measured deterministically.

## Root cause

`docs/index.md` embeds the benchmark chart with no reserved dimensions:

```html
<p align="center">
  <img alt="Shows a bar chart with benchmark results." src="...#only-light">
</p>
```

The `<img>` has no `width`, `height`, or `aspect-ratio`. Before the image bytes arrive the
browser reserves **zero** vertical space for it, so the moment it loads, every element below
it (the caption, the "Highlights" heading, the feature list) is pushed down — a visible
"jump" and a Core Web Vitals **CLS** hit.

## Files

| File | Purpose |
| --- | --- |
| `homepage-repro.html` | The homepage markup as-is — dimensionless `<img>` (reproduces the bug). |
| `homepage-fixed.html` | Same markup with the proposed fix (`aspect-ratio` reserves the box). |
| `benchmark.svg` | Stand-in benchmark chart (1000×300) so the repro is offline & deterministic. |
| `measure-cls.mjs` | Playwright script: delays the image, measures CLS via the Layout Instability API, saves before/after screenshots. |
| `results.json` | Captured measurement output (regenerated by the script). |
| `screenshot-*-before.png` / `screenshot-*-after.png` | Visual proof of the shift (and its absence after the fix). |

## How to run it

```bash
npm init -y
PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1 npm i playwright   # uses your local Google Chrome
node measure-cls.mjs
```

(Or simply open `homepage-repro.html` in a browser with the Network tab throttled to "Slow
3G" and watch the page jump when the chart appears; do the same with `homepage-fixed.html`
to see it stay still.)

## Measured result

CLS measured with the W3C Layout Instability API — the same metric Chrome DevTools and
Lighthouse report. Lighthouse rates CLS **> 0.1** as "needs improvement".

| Viewport | Page | Image height before → after | Layout-shift events | **CLS** |
| --- | --- | --- | --- | --- |
| Desktop 1280×800 | buggy (`homepage-repro.html`) | 0px → 221px | 1 | **0.046** |
| Desktop 1280×800 | fixed (`homepage-fixed.html`) | 221px → 221px | 0 | **0** |
| Mobile 390×844 | buggy | 0px → 107px | 1 | **0.049** |
| Mobile 390×844 | fixed | 107px → 107px | 0 | **0** |

**Verdict:** the dimensionless image shifts content downward on load (CLS > 0); reserving the
image's aspect ratio removes the shift entirely (CLS = 0). The measured CLS here is a *lower
bound* — it only models the top of the page; the live docs page has more content below the
image and uses a larger benchmark image, so the real-world shift is at least this large.
